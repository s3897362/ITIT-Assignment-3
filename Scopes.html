Scopes and Limits

Scope

This project was initiated to create a synthesiser that could create a unique sound set on a low budget device and being able to increase the availability of this product to many other users with many unique sounds played through the synthesiser. The main objective for this project is to create a synthesiser soundboard that outputs sound according to the unique footprint of a synthesiser like serial numbers and unique FTDI port identifiers. The Arduino synthesiser will be programmed with C#. The deliverable outcomes of this project are the tangible Arduino synthesiser that users can press the button to listen to a specific sound, for now we are using the serial number to generate wave tables and frequencies using the Mozzi sonification library. With ideally some potentiometer to modify our sound with board defined parameters.

Limits

The main idea of this project is to develop a synthesizer which creates a new unique sound via electronic equipment and sensors for a low cost. This synthesizer was planned to have multiple features that realistically cannot be completed in the time frame that was given to us. The basic prototype would be created within the timeframe that was given composing of a single feature within the synthesizer being able to create sounds which are generated through the Arduino Uno's serial number which would give unique readouts that are translated into unique sounds that are played through the speaker. The limitation of this timeframe would mean that the project would not be completed to its full potential and that there will be some areas for improvement. The use of this one model is to show the proof of concept that unique sounding music can be created without the need for expensive equipment. This synthesizer would have some possible limits including the time we have to learn and implement code required to run the synthesizer on the Arduino uno board. More limits would include the ability to get all the parts required to complete the project. The project would not have every part required for all of its features due to the time constraints and monetary budgets. However certain features that won’t be involved would be certain sounds that a user would like to be played but it unfortunately won’t work because of the programmed sound that was chosen from the library, other sensors that won’t be incorporated in the synthesiser due to the lack of time and resources needed to implement this feature. Although the prototype should complete basic functionality and perform as per described.
