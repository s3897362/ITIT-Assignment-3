Testing

We will be testing the synthesiser by first seeing if we can code it correctly. This would mean being able to code the Arduino for the synthesiser. To test the Arduino would just run the code, if it doesn't work then we would need to fix the code. Once we have successfully coded what we need in the Arduino, we then create the actual synthesiser. Once the synthesizer is made and the Arduino hooked up, we can test it. We would first test it by pressing the key or keys which we have made and seeing if it produced a sound, if it does then that means the key is working correctly. Once that is done we will test the location sensor. We will do this by playing the key in different locations and seeing if the sound/noise made from pressing the key changes. We will know when we have succeeded, when the key works and that the sound changes depending on where we are playing it.The Arduino IDE also has a verify function which can be used for testing and debugging, which provides an error list with any logical errors contained in the code. These errors can then be reviewed and debugged for faults and syntax and or logical errors. Furthermore these errors can be fixed by identifying the error, creating a plan on how to solve the issue, writing the code again using pseudocode and finally applying the pseudocode and correcting the errors made.
